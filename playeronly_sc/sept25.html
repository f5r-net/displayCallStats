Incidents and new issues

  • [RTC] For the iOS 4.1.1.x version, using Xcode 16 to build and distribute the app to the App Store may result in the error message: “Asset validation failed: Invalid Executable. The executable contains bitcode”
  • There is a workaround for it which is to execute the handle_bitcode.sh to remote the bitcode before compiling or building the project.
  • [RTC] For iOS SDK version 4.2.6 and above, calling API setBeautyEffectOptions will trigger a permission request window on the audience's side.
  • The fix is merged in SDK versions 4.5.0.
  • [RTC] For Android 4.1.1.25 and above, the camera focus fails to work after switching the capture content between close objects (e.g., a human face) and far objects (e.g., text on a screen) multiple times
  • Using private API che.video.android_camera_auto_faceDetect_focus=false can resolve this issue. This private API will turn off auto_facedetect_focus.
